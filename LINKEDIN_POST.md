## Option 4: Technical Deep-Dive

⚡ **Deep Dive: Building a Production-Grade CSP-based Timetable Scheduler**

**Problem Space:**
University timetabling is an NP-complete problem requiring simultaneous optimization of:
• Instructor availability constraints
• Room-course type compatibility (labs vs lectures)
• Time slot allocation
• Resource distribution balancing

**Technical Implementation:**

**1. Algorithm Evolution:**
• Started with simple greedy CSP → One class per instructor bug
• Enhanced with balanced distribution → Still had the bug
• Developed Fixed CSP → Multiple classes per instructor ✅

**2. CSP Solver Features:**
• Minimum Remaining Values (MRV) heuristic
• Forward checking for early pruning
• Conflict detection with O(1) lookup using hash sets
• Randomization for fair resource distribution

**3. Data Structure Optimization:**
Changed from:
`Dict[instructor_id → single_assignment]`

To:
`List[(instructor_id, timeslot, room, course)]`

This simple change unlocked multiple assignments per instructor!

**4. Architecture:**
• Modular design: CSP solver, GUI, data loaders, exporters
• Observer pattern for progress tracking
• Strategy pattern for algorithm selection

**5. Performance:**
• 100+ instructors: 8-15 seconds
• 200+ instructors: 15-30 seconds
• Memory-efficient: Handles 500+ instructors

**Results:**
Production-ready system with beautiful GUI, multiple export formats, and intelligent scheduling.

**Stack:** Python 3.14 | CSP Algorithms | Tkinter | Pandas

Code demonstrates: Algorithm design, software architecture, UI/UX, and problem-solving!
